# 反復的検索パターン

サブエージェントに渡すべきファイルを最初から正確に指名するのは難しい。広いクエリで候補を集め、関連性を評価し、クエリを絞って繰り返す。**Glob / Grep / Read を組み合わせる Claude Code の検索手順**を 4 フェーズに分解する。

## 解決する問題

サブエージェント（または自分自身）に渡すコンテキストを決める時の典型的な失敗:
- **全部送る**: コンテキスト枠を超え、ノイズで埋もれる
- **何も送らない**: 重要ファイルを欠いたまま作業が始まる
- **推測で送る**: コードベースで実際に使われている用語と違っていて空振りする

## 4 フェーズ

```
DISPATCH → EVALUATE → REFINE → LOOP（最大3サイクル）
```

### フェーズ1: DISPATCH — 広く拾う

最初は**意図ベースの広いキーワード**で検索する。

具体ツール:
- `Glob` で対象パスをざっくり絞る（例: `src/**/*.ts`）
- `Grep` で複数キーワードを OR 検索
- controller として動く場合に限り、必要なら `Agent (subagent_type=Explore)` に「○○に関連するファイルを探す」と依頼する。read-only サブエージェント（explorer 等）は委譲・spawn せず `Grep`/`Glob`/`Read` で進める

```bash
# 例: 「認証トークン期限切れバグ」を直す前のフェーズ1
# → 直接コマンドではなく Glob+Grep ツールで実行
Grep: pattern="token|auth|expir" path="src" output_mode="files_with_matches"
```

このフェーズの出力: **候補ファイルパスのリスト**（10〜30件くらい想定）。

### フェーズ2: EVALUATE — 関連性を採点

候補を読み、各ファイルに 0〜1 の関連スコアを内部的に付ける。

判定基準:
| スコア | 意味 |
|---|---|
| 0.8〜1.0 | 対象機能を**直接実装**している |
| 0.5〜0.7 | 関連型・パターン・呼び出し元を含む |
| 0.2〜0.4 | 間接的に関連（共通ユーティリティ等） |
| 0〜0.2 | 無関係 |

実行手順:
1. 上位候補を `Read` で開く（最初は冒頭 100〜200 行で十分なことが多い）
2. 「このファイルが対象タスクの中心か / 周辺か / 無関係か」を判定
3. 同時に**コードベースで実際に使われている用語**をメモする（次のフェーズで使う）
4. **欠けているコンテキスト**を列挙する（「ルーター設定が見当たらない」「ミドルウェア層がまだ見えない」等）

### フェーズ3: REFINE — クエリを絞り直す

評価結果からクエリを更新する:
- **追加するキーワード**: 高関連性ファイル中で実際に使われていた用語（推測語ではなく実物）
- **追加する除外パス**: スコア 0.2 未満が確定したディレクトリ
- **次に狙う領域**: 「欠けているコンテキスト」で挙がった層

実例:
```
初期キーワード: ["rate", "limit"]
→ 候補ヒットなし
→ コードベースで使われている用語は "throttle"（middleware で発見）
REFINE: ["throttle", "middleware"] に置換
```

### フェーズ4: LOOP — 最大3サイクルで停止

各サイクル後に「停止条件」を確認:
- 関連性 0.7 以上のファイルが **3〜5 件**揃った
- 「欠けているコンテキスト」リストが空、または既知のものだけ

満たせば停止。満たせなくても**3 サイクルで打ち切る**（4 サイクル目以降は得るものが少ない）。

## 実例

### 例1: バグ修正のコンテキスト収集

```
タスク: 「認証トークンの有効期限バグを修正」

サイクル1:
  DISPATCH: Grep "token|auth|expir" src/
  EVALUATE: auth.ts (0.9), tokens.ts (0.8), user.ts (0.3)
  REFINE: +"refresh" +"jwt"、user.ts の親ディレクトリは除外候補

サイクル2:
  DISPATCH: Grep "refresh|jwt" + auth/ サブツリー
  EVALUATE: session-manager.ts (0.95), jwt-utils.ts (0.85)
  → 高関連性 4件、停止

返すコンテキスト: auth.ts, tokens.ts, session-manager.ts, jwt-utils.ts
```

### 例2: 機能実装のコンテキスト収集

```
タスク: 「APIエンドポイントにレート制限を追加」

サイクル1:
  DISPATCH: Grep "rate|limit" routes/
  EVALUATE: ヒットほぼなし。コードベースは "throttle" 用語
  REFINE: ["throttle", "middleware"] に置換

サイクル2:
  DISPATCH: Grep "throttle|middleware"
  EVALUATE: throttle.ts (0.9), middleware/index.ts (0.7)
  → ルーター層が見えていない（gap）
  REFINE: +"router" +"express"

サイクル3:
  DISPATCH: Grep "router|express"
  EVALUATE: router-setup.ts (0.8)
  → 高関連性3件、停止

返すコンテキスト: throttle.ts, middleware/index.ts, router-setup.ts
```

## サブエージェント呼び出しに組み込む

（このセクションは controller 向け。read-only サブエージェントとして動く場合（例: 本リポジトリの `explorer`）は `Agent` での委譲・spawn を行わず、`Grep`/`Glob`/`Read` のみでこの 4 フェーズを実行する。spawn は controller のみ。

なお本セクションの `subagent_type=Explore` は Claude Code 組み込みの読み取り専用探索エージェント `Explore` を指し、本リポジトリのループ役 `explorer`（小文字）とは別物。広範な探索の委譲先として組み込み `Explore` を使う。）

controller がサブエージェントを `Agent` ツールで起動する**前**にこの 4 フェーズを回し、決まったファイルを prompt 内で**明示**して渡す:

```
プロンプトに含める形:
- 「以下のファイルが関連します: auth.ts:42-80, tokens.ts:1-50」
- 「無関係と判定済み: user.ts, profile.ts」
```

サブエージェントに「自分で探せ」と任せると、再度フェーズ1から始まり tool_uses が膨れる。**親で絞ってから渡す**。

## 境界条件 / 未定義時の運用

実運用で曖昧になりやすい点を先に決めておく:

- **ファイル全体が短い時の Read**: 全体が 200 行未満なら `limit` を省略して全文読む。「冒頭 100〜200 行」は大規模ファイルへのデフォルトであって上限ではない
- **除外パスの粒度**: 同一ディレクトリ内で 2 ファイル以上が確定で 0.2 未満なら**親ディレクトリ単位**で除外、それ以外はファイル単位で除外。粒度を混ぜるなら理由を明示
- **ヒット過多時の絞り込み閾値**: Grep のヒットが 30 件超 or 5 ディレクトリ以上に分散したら Glob でパス絞り込みに進む。それでも解消しないなら次サイクルで除外条件を強める
- **フェーズ1の DISPATCH を Agent に委譲する判断**: 委譲は controller のみが行う。探索範囲がプロジェクト全体（ファイル数千以上）かつ用語の手がかりが弱い場合のみ controller が `Agent (Explore)` に投げる。範囲が `src/` 配下程度、または自身が read-only サブエージェントの場合は委譲せず `Grep`+`Glob` で進める

## ベストプラクティス

1. **広く始めて、段階的に絞る** — 初期クエリを過度に特定しない
2. **コードベース実物の用語を学ぶ** — 1サイクル目はほぼ用語学習。推測キーワードは多くが空振りする
3. **欠けているものを言葉にする** — 暗黙のギャップは消えない。明示すると次のクエリが定まる
4. **「3〜5件揃ったら十分」** — 中スコア10件より高スコア3件
5. **自信を持って除外する** — 低スコアと判定したパスは次サイクル以降ほぼ覆らない
6. **3サイクルで打ち切る** — 4サイクル目以降は ROI が急落する。残ギャップは subagent 側で埋めさせる

## 関連

- `Agent` ツール（subagent_type=Explore） — フェーズ1の DISPATCH 委譲先（controller 専用。explorer 等の read-only サブエージェントは使わない）
- `Grep` / `Glob` / `Read` — このスキルが組み立てる基本ツール
- `.claude/agents/` — このパターンを内部的に使うエージェント定義
