# 横断フィードバック索引

特定 work item に閉じず、以後の作業全般に効く人間フィードバックの索引。運用（継続的改善ループ・エントリ形式・`種別` の定義）は `$local-memory` skill の「横断フィードバック」節を参照する。

- 詳細エントリは scope 別ファイルに置く: `review.md` / `implementation.md` / `research.md` / `general.md`。
- 各エージェントは「自分の scope ＋ general」だけ読む（phase-reviewer・verifier → review、implementer → implementation、tech-docs-searcher → research、全員 → general）。
- 追記・status 更新は controller のみ。`retired` は各ファイル末尾の `## アーカイブ` へ移す。
